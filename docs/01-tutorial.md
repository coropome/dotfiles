# Tutorial（5〜10分）: iTerm2 + tmux を「最初から」使えるようになる

目的：このチュートリアルが終わったら、最低限
- セッションを作る/戻る
- 画面を分割して移動する
- 困ったらヘルプを見る
ができるようになる。

## 0. 準備

```bash
make install
```

- `make install` で Homebrew の core パッケージ導入と tmux / zsh / git の基本設定をまとめて入れる。

## 1. tmuxを起動する（いつもの入口）

```bash
tnew
```

- ここから先は、**tmuxの中**。
- 「戻りたい」時は `Ctrl-a d`。

## 2. まずヘルプを見る

- ミニヘルプ: `Ctrl-a ?`
- フルヘルプ: `Ctrl-a /`

> 困ったらまず `Ctrl-a ?`。

## 3. 画面を分割する

- 縦分割（左右に割れる）: `Ctrl-a |`
- 横分割（上下に割れる）: `Ctrl-a -`

分割したら、次へ。

## 4. 分割した画面を移動する

- 左: `Ctrl-a ←`
- 下: `Ctrl-a ↓`
- 上: `Ctrl-a ↑`
- 右: `Ctrl-a →`

まずは「どこにいるか」より、**押したら動く**体験を優先。

## 5. 退出（detach）→ 戻る

- tmuxから抜ける: `Ctrl-a d`
- もう一回戻る: `tgo`（選べる） or `tnew`（新規/attach）

## 6. つまずきチェック

- 「tmuxから抜けられない」→ `Ctrl-a d`
- 「どのセッションに戻ればいい？」→ `tlist` → `tgo`
- 「キー忘れた」→ `Ctrl-a ?`


## 7. スクロール＆コピー（コピーモード）

ログが流れて選択しづらいときは、tmuxの**コピーモード**が便利。

### 入り方
- `Ctrl-a [`（tmux標準）

### 最小で覚える（これだけ）

**スクロールして読む**
- 下/上：`j` / `k`
- 半ページ：`Ctrl-d` / `Ctrl-u`
- 検索：`/` → Enter（次へ：`n`）

**範囲コピー（システムクリップボードへ）**
- `Space`：選択開始
- 移動（`h/j/k/l` か矢印）で範囲を伸ばす
- `Enter`（または `y`）：コピーして終了

※ iTerm2の通常コピー（範囲選択→Cmd+C）で十分な間は、無理にコピーモードを常用しなくてOK。

> ditfilesでは、コピーモードの `Enter` / `y` でシステムクリップボードへコピーできるようにしてある。macOS では `Cmd+V`、Linux / WSL では OS 側の paste を使う。詳細は `docs/11-copy-mode.md`。


## 8. 貼り付け（tmuxバッファ）

- 最後にコピーしたものを貼り付け：`Ctrl-a p`（互換: `Ctrl-b p`）
- 以前コピーしたものを選ぶ：`Ctrl-a b`（互換: `Ctrl-b b`）

※ 普段は OS 側のクリップボード貼り付けで十分。tmux内で完結したい時だけ使う。
